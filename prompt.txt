社区垃圾分类系统，以下是详细的技术和功能要求：

### 一、技术框架
1. **前端**
    - **技术选型**：采用 JSP（JavaServer Pages）进行页面开发。利用 JSP 的动态网页技术，实现与后端数据的交互和展示。确保页面设计简洁、易用，符合社区居民的操作习惯。
    - **页面布局与样式**：设计合理的页面布局，包括导航栏、内容区域、页脚等。使用 CSS（Cascading Style Sheets）进行样式设计，保证页面的美观和响应式设计，能够适配不同屏幕尺寸，如电脑、平板和手机。
2. **后端**
    - **框架选择**：基于 Java 的 Spring Boot 框架构建后端服务。利用 Spring Boot 的自动配置、起步依赖等特性，快速搭建稳定、高效的后端应用。遵循 RESTful API 设计规范，提供清晰、标准的接口与前端进行交互。
    - **服务分层架构**：采用分层架构，如 Controller 层负责接收前端请求并返回响应，Service 层处理业务逻辑，Repository 层负责与数据库交互。确保各层职责明确，代码结构清晰，易于维护和扩展。
3. **数据库**
    - **数据库选型**：选用 MySQL（mybatis） 作为数据库管理系统，用于存储系统中的各类数据。根据系统功能需求，合理设计数据库表结构，确保数据的完整性、一致性和高效存储。

### 二、功能模块
1. **用户注册与登录**
    - **用户注册**
        - **注册页面设计**：创建一个注册页面，包含必填的基本信息字段
        - **注册逻辑**：当用户点击注册按钮，后端接收并验证数据无误后，将用户信息加密存储到数据库中，密码采用md5算法进行加密。
    - **用户登录**
        - **登录方式**：支持账号+密码登录和手机+密码登录两种方式。
        - **账号+密码登录**：创建登录页面，用户输入用户名和密码，后端验证用户名是否存在以及密码是否正确。
        - **手机+密码登录**：创建登录页面，用户输入手机号和密码，后端验证手机号是否存在以及密码是否正确。
2. **垃圾分类指南**
    - **分类查询**
        - **数据展示**：在页面上以列表或卡片形式展示可回收物、厨余垃圾、有害垃圾、其他垃圾四大类别的详细信息。对于每一类垃圾，除了提供文字说明该类别垃圾的定义、包含的常见物品示例外，还需展示至少 3 张相关的图片，图片需清晰展示该类别垃圾的特征。
        - **搜索功能**：添加搜索框，用户可以输入垃圾名称，系统快速定位并展示对应的分类信息，提高查询效率。
    - **常见问题**
        - **问题与解答管理**：在数据库中创建常见问题表，存储居民可能遇到的垃圾分类问题及对应的解答。管理员可在后台对问题和解答进行添加、修改和删除操作。
        - **前端展示**：在前端页面以列表形式展示常见问题，用户点击问题可展开查看详细解答。
3. **垃圾投放记录**
    - **手动记录**
        - **记录页面设计**：设计一个垃圾投放记录页面，居民登录后可在此页面手动输入垃圾投放信息。包含投放类型（从垃圾分类的四大类别中选择）、投放时间（提供日期和时间选择器，方便居民准确选择投放时间）等字段。
        - **数据保存**：居民提交记录后，后端将数据保存到数据库中，同时记录用户的唯一标识，以便后续统计和查询。
    - **投放提醒**
        - **提醒设置**：在系统后台设置每周固定的垃圾投放提醒时间，如每周一、周三、周五的上午 8 点。
        - **提醒实现**：利用定时任务（如 Spring Boot 中的 @Scheduled 注解），在设定的提醒时间，通过消息推送（如 WebSocket 实现实时推送，或者短信推送）向居民发送垃圾投放提醒消息。提醒消息内容需简洁明了，告知居民当天是垃圾投放时间。
4. **社区公告**
    - **通知发布**
        - **发布权限**：只有社区管理者（具有特定权限标识的用户）能够登录系统后台发布通知。
        - **发布页面设计**：在后台管理系统中创建通知发布页面，包含标题（长度限制在 20 - 50 个字符之间）、内容（支持富文本编辑，方便管理者添加图片、链接等）、发布时间（自动获取当前系统时间）等字段。
        - **数据存储与展示**：发布的通知保存到数据库中，前端社区公告页面按照发布时间倒序展示通知列表，用户点击通知标题可查看详细内容。
    - **活动宣传**
        - **活动信息管理**：社区管理者在后台创建活动宣传页面，录入活动名称、活动时间、活动地点、活动内容简介、报名方式（如链接、电话等）等信息。活动信息同样存储在数据库中。
        - **前端展示**：在前端社区公告页面以特定区域展示活动宣传信息，采用醒目的样式突出显示，吸引居民关注。活动信息按照活动时间先后顺序展示，对于已过期的活动，进行特殊标识或隐藏处理。
5. **数据统计（仅超级管理员可用）**
    - **居民参与统计**
        - **注册数量统计**：提供一个统计页面，超级管理员登录后可查看系统中居民的注册总数。通过查询用户表记录数实现。
        - **活跃度统计**：定义活跃度计算规则，例如根据用户登录次数、垃圾投放记录次数等指标综合计算。在统计页面展示活跃用户数量（过去一周内有操作记录的用户）以及活跃度趋势图（以周为单位，展示过去 12 周的活跃度变化）。
    - **垃圾投放统计**
        - **分类统计**：统计每种垃圾类型的投放次数、投放重量（若有相关记录）等信息。在页面上以图表形式（如柱状图、饼图）展示各类垃圾的投放占比情况，直观呈现居民的垃圾分类情况。
        - **时间维度统计**：提供按日、周、月等时间维度统计垃圾投放记录的功能。例如，展示本周每天各类垃圾的投放数量变化曲线，帮助分析居民垃圾投放行为随时间的变化规律。
    - **数据导出**
        - **导出功能实现**：在统计页面提供导出按钮，超级管理员点击按钮可将当前展示的统计数据导出为 Excel 表格。导出的数据需包含清晰的表头，数据格式规范，便于进一步分析。
6. **管理员功能**
    - **用户管理**
        - **用户查询**：在后台管理系统中创建用户管理页面，管理员可根据用户名、手机号、邮箱等条件进行单个或批量查询用户信息。查询结果以表格形式展示，包含用户基本信息和注册时间等字段。
        - **用户添加**：提供添加用户的功能入口，管理员可手动输入用户信息完成添加操作。输入字段与用户注册页面类似，但需注意密码的加密处理。
        - **用户修改**：管理员在用户列表中选择需要修改的用户，进入修改页面，可对用户的基本信息进行修改，修改后的数据同步更新到数据库。
        - **用户删除**：支持管理员在用户列表中单个或批量删除用户，删除操作需进行二次确认，确保数据安全。
    - **公告管理**
        - **公告发布**：同社区公告 - 通知发布中的发布功能，管理员可在后台发布新的社区公告。
        - **公告编辑**：对于已发布的公告，管理员可在后台找到对应的公告记录，进入编辑页面修改公告标题、内容等信息，修改后的数据实时更新到前端展示页面。
    - **数据查看**
        - **垃圾投放记录查看**：管理员可在后台查看所有居民的垃圾投放记录，支持按用户、时间、垃圾类型等条件进行筛选查询。记录以表格形式展示，包含投放用户、投放时间、投放类型等详细信息。
        - **统计数据查看**：管理员能够查看与超级管理员相同的居民参与统计和垃圾投放统计数据，但无数据导出权限。在页面展示形式上，与超级管理员的数据统计页面保持一致。
